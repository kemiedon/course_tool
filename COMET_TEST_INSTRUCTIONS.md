# Comet 瀏覽器測試指令

## 前置準備

1. 啟動開發伺服器：

```bash
npm run dev
```

2. 在 Comet 瀏覽器中開啟應用程式：

```
http://localhost:5173
```

---

## 測試案例 1：基本資訊設定（Step 1）

**測試資料（來自 TEST_DATA.md）：**

- 課程主題：打造 AI 自學力，用 Gemini 3+ NotebookLM 讓孩子自主學習
- 課程類別：兒童青少年
- 時數：10 小時
- 每次上課時數：2 小時
- 開課日期：2026-01-26
- 上課時段：週一到週五 09:10-11:10

**Comet 指令：**

```
請幫我填寫課程基本資訊表單：
1. 課程主題填入「打造 AI 自學力，用 Gemini 3+ NotebookLM 讓孩子自主學習」
2. 課程類別選擇「兒童青少年」
3. 總時數填入「10」
4. 每次上課時數填入「2」
5. 開課日期選擇「2026-01-26」
6. 勾選週一、週二、週三、週四、週五
7. 上課時間填入「09:10-11:10」
8. 點擊「AI生成課程名稱」按鈕
9. 等待 AI 生成課程名稱完成（約 5-10 秒）
10. 點擊「下一步」按鈕
11. 截圖保存為「step1-basic-info.png」
```

**預期結果：**

- 課程排程表顯示 5 天課程（週一到週五）
- 每天顯示日期、時間「09:10-11:10」、時數「2 小時」
- 總時數為 10 小時
- AI 生成的課程名稱出現在輸入框中

## 測試案例 2：課綱生成與複製（Step 2）

**Comet 指令：**

```
請幫我測試課綱生成功能：
1. 在課綱編輯器中，點擊第一天的「AI生成課綱」按鈕
2. 等待約 10-15 秒讓 AI 生成課綱內容
3. 確認課綱內容已顯示在卡片中
4. 點擊第一天課綱卡片右上角的「複製」按鈕（📋 圖示）
5. 確認出現「✅ 課綱已複製到剪貼簿！」的 toast 通知
6. 繼續點擊其他天數的「AI生成課綱」按鈕
7. 等待所有課綱生成完成
8. 點擊頁面上方的「複製全部課綱到剪貼簿」按鈕
9. 確認出現「✅ 所有課綱已複製到剪貼簿！」的 toast 通知
10. 截圖保存為「step2-curriculum.png」
11. 點擊「下一步」按鈕
```

**預期結果：**

- 每天的課綱都成功生成（包含學習目標、課程內容、教學活動）
- 單個複製功能正常運作並顯示 toast 通知
- 全部複製功能正常運作並顯示 toast 通知
- 課綱內容以 Markdown 格式渲染

## 測試案例 3：宣傳文案生成（Step 3）

**Comet 指令：**

```

1. 在宣傳編輯器中，點擊「AI生成宣傳內容」按鈕

7. 截圖保存為「step3-promotion.png」
8. 點擊「下一步」按鈕
```

**預期結果：**

- AI 生成的宣傳文案約 200 字
- 文案基於課綱重點撰寫
- 文案針對目標受眾的痛點（兒童青少年：學習動力、實用技能）

## 測試案例 4：資訊圖表生成（Step 4）

**Comet 指令：**

```
請幫我測試資訊圖表生成功能：
1. 在資訊圖表生成器中，點擊第一天的「生成圖片」按鈕
2. 等待約 20-30 秒讓 Imagen 3.0 生成圖片
3. 確認圖片成功顯示（16:9 比例的路線圖風格）
4. 確認圖片風格為「可愛卡通風格」（因為課程類別是兒童青少年）
5. 確認圖片包含：
   - 左側面板（約 20%）：學習目標說明
   - 中央路線圖（約 60%）：7 個階段的時間軸
   - 右側面板（約 20%）：關鍵亮點
6. 點擊「上傳圖片」按鈕（如有需要）
7. 繼續生成其他天數的圖片
8. 截圖保存為「step4-infographics.png」
9. 點擊「儲存課程」按鈕
```

**預期結果：**

- Imagen 3.0 成功生成 16:9 比例的路線圖風格圖片
- 圖片風格符合「兒童青少年」類別（可愛卡通風格）
- 圖片包含完整的路線圖布局（左、中、右三個區塊）
- 圖片可以成功上傳到 Firebase Storage

## 注意事項

1. **API 金鑰設定**：測試前請確認 Firebase 和 Google AI 的 API 金鑰已正確設定
2. **網路連線**：確保網路連線穩定，因為 AI 生成需要呼叫外部 API
3. **等待時間**：AI 生成需要時間，請耐心等待：
   - 課程名稱生成：約 5-10 秒
   - 課綱生成：約 10-15 秒/天
   - 宣傳文案生成：約 10-15 秒
   - 圖片生成：約 20-30 秒/張
4. **截圖位置**：所有截圖會保存在專案根目錄
